Data handling

What Nova reads, writes, and stores.

/ READS

From your PSA, on a schedule

Ticket subject, body, status, priority, contact. Read-only API access via your Autotask user. We see what your dispatcher sees — nothing more.

/ WRITES

When dispatcher clicks Accept

Updated classification fields (priority, queue, category, work type). Internal notes for transparency. Time entries when techs check out from Field Ops. All changes logged.

/ STORES

Only what we need to operate

Ticket metadata (IDs, classifications, history) and learned rules. Encrypted at rest (AES-256) and in transit (TLS 1.3). Ticket bodies are not persisted — read, classified, discarded.

/ AI usage

Your tickets are not training data.

Nova uses Anthropic's enterprise API with zero-retention enabled. Prompts are processed and discarded — not stored, not used to train models, not visible to anyone at Anthropic outside of your active inference call.

  • Anthropic enterprise API · zero data retention
  • Prompts redacted of obvious PII before sending
  • Optional: bring your own AI provider with self-hosted compliance

Access & auth

Who can see what.

Inside your MSP

  • Microsoft Entra SSO required for all staff access
  • Role-based: dispatcher, tech, manager, owner, billing
  • Group-based provisioning — Entra group changes propagate
  • Audit log of every classification, override, login
  • Session timeout configurable (default 8 hours)

Inside your client portal

  • SSO via M365, Google, or SAML 2.0
  • Per-client roles: regular user, manager, billing, admin
  • Org-level isolation: clients see only their own tickets
  • Public/private ticket comment separation
  • Optional MFA enforcement per client policy
