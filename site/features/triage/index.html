How it works

Two AI passes. One dispatcher in the loop.

PASS 01

Sonnet classifies fast

Claude Sonnet reads ticket subject, body, and contact context. Returns priority, issue type, queue, category, sub-category, work type, source — plus a confidence score per field.

PASS 02

Opus verifies low-confidence

Anything below the auto-accept threshold gets a second pass with Claude Opus. Fewer false dispatches, better edge-case handling, conditional cost.

REVIEW

Dispatcher decides

Above 85% confidence: auto-accept. Below: lands in review with reasoning. One click to approve or override. Every override trains the next pass.

/ Rules engine

96 rules and counting. Every override teaches the model.

Nova starts with 18 pattern rules — common ticket shapes (password resets, printer jams, M365 license adds). As your dispatcher overrides AI calls, those become learned rules. After 90 days you typically have 70+ learned rules tuned to your client base.

  • 78 learned rules from production overrides
  • 18 pattern rules shipped out of the box
  • Rules sync across all your queues — one client teaches all
  • Inspect, edit, or pause any rule from the admin console
/ Confidence + thresholds

Per-queue auto-accept thresholds. You stay in control.

High-volume routine queues (password resets, license requests) auto-accept at 85% confidence. Sensitive queues (security incidents, P1s) stay at 100% — every call goes through dispatch review. Tune any threshold per queue from the admin panel.

  • Auto-accept threshold per queue, per client tier
  • Reasoning string on every classification
  • Audit log: who accepted, who overrode, when, why
  • Daily clean-rate report posted to Teams
